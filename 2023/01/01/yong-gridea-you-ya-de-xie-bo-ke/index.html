最近在家有点无聊，便寻思着写点博客。在网上找了挺多教程，但随着不断的寻找教程，
那些教程虽然最后的博客很漂亮，但似乎丧失了博客的本质，于是我就发现了他：Gridea。

在开始前的一段话
Gridea官方网站: https://www.gridea.dev，去下载符合自己系统的版本
本文章建议有Github使用经验的人使用，没有基础的可以去网上查一查相关名词

开始!
一个Github账号，符合自己系统的Gridea软件
由于官方wiki上写到了Gridea的安装，也是十分简单，这里就不过多叙述

注册GitHub账号
进入网站:https://www.github.com，注册的教程是个人都会，如果进不去网站，可以科学上网或修改hosts文件（以后考虑发一篇博客）

搭建仓库
注册后点击右上角加号，新建仓库，仓库名 用户名.github.io 用户名写自己注册时的用户名，并添加README文件

创建token
点击 「右上角自己的头像右边的小箭头」→ 「Settings」→ 「Developer settings」
再选择带有钥匙标志的一项，新建token，名字随便起，一定要选择repo！repo要全选，关键！！！
最后最重要的一项：一点要记住你的token密码，它只会显示一次
申请成功，会给你一串数字 + 字母的信息，这个就是 Token，我们一定要保存好！后面的配置中我们会用到。

配置Gridea
进入软件，右下角选择远程，填写内容
域名：Github Pages 对应的域名（例如：https://jiangmingce.github.io）
仓库：你的静态文件存放的仓库（例如：http://jiangmingce.github.io ）
分支：你的 Github Pages 对应的分支（应该是main）
用户名：仓库所属用户的用户名（通常就是你的 Github 账号的昵称，例如：jiangmingce）
邮箱：Git 推送时使用的邮箱（通常就是你的 Github 账号的邮箱，例如：jmc.lu@foxmail.com ）
Token: Git 推送时需要的 Token，用来向 Github 提交构建后的文件 这里的 token 就是 申请的 Github personal access tokens
CNAME: 可通过这个【可选项】配置你自己的域名(一般不用)

填写后，点击右下角「检测远程连接」 按钮，如果提示 「远程连接成功」 那就 ok 啦～
这时候，你用浏览器输入你的网址（就是你刚刚填写的Github域名），就可以访问啦

最后
Gridea的功能不仅那么点，这是只是配置了部分内容，后续考虑发布教程去完善，大家也可以查看wiki
